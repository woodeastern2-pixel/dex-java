# 데이터 안전 및 보안

개정일: 2026년 5월 18일
시행일: 2026년 5월 18일

## 1. 기본 원칙

루미는 필요한 데이터를 가능한 한 사용자의 단말 내부에서 처리하고, 원격 기능은 사용자가 설정했거나 기능 실행에 필요한 경우에만 사용합니다.

## 2. 로컬 저장

- 대화 기록, 루미의 상태, 기억, 친밀도, 성장 데이터는 앱 내부 데이터베이스에 저장됩니다.
- API 키와 민감 설정은 가능한 경우 Android Keystore 기반 암호화 저장소에 보관합니다.
- 단말의 보안 저장소가 손상되었거나 사용할 수 없는 경우 앱 전용 저장소로 폴백할 수 있으므로, 사용자는 단말 잠금과 OS 보안 업데이트를 유지해야 합니다.

## 3. 백업 제외

루미는 Android 백업 및 기기 간 전송에서 다음 항목을 제외하도록 설정합니다.

- 대화 데이터베이스
- 루미 상태·기억 데이터
- 일반 설정 및 동의 상태
- API 키 저장소
- 음성 관련 파일 경로

단, 사용자가 직접 내보낸 대화 파일이나 갤러리에 저장한 이미지는 사용자가 선택한 저장 위치의 백업 정책을 따릅니다.

## 4. 네트워크 전송

- 원격 LLM, 이미지 생성, 웹 검색, 음성 인식 기능을 사용할 때 외부 서비스로 데이터가 전송될 수 있습니다.
- 외부 통신은 HTTPS 기반 API를 사용합니다.
- 외부 제공자의 보관 기간, 학습 사용 여부, 삭제 정책은 각 제공자의 정책에 따릅니다.

## 5. 권한 사용

- 인터넷: 원격 AI/API, 이미지 생성, 웹 검색에 사용합니다.
- 마이크: 음성 입력을 텍스트로 변환할 때 사용합니다.
- 알림: 루미의 능동 대화 알림을 표시할 때 사용합니다.
- 부팅 완료 수신: 단말 재시작 후 능동 대화 예약을 복원할 때 사용합니다.

## 6. 콘텐츠 안전

- 이미지 생성에는 NSFW 필터 옵션을 제공합니다.
- 필터를 끄더라도 외부 API 제공자의 자체 안전 정책이 적용될 수 있습니다.
- 사용자는 불법·유해·타인 권리 침해 콘텐츠를 입력하거나 생성 요청해서는 안 됩니다.

## 7. 사용자 보호 권장 사항

- 실제 API 키를 공유하거나 스크린샷으로 노출하지 마세요.
- 민감한 개인정보, 금융정보, 인증번호, 비밀번호를 대화에 입력하지 마세요.
- 중요한 대화 내역을 내보낸 경우 파일 저장 위치와 공유 대상을 직접 관리하세요.
- 공용 또는 분실 위험이 있는 단말에서는 단말 잠금과 앱 데이터 삭제 기능을 함께 사용하세요.
